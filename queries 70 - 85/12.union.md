# ✅ **10 Exercícios com UNION / UNION ALL**

---

### **1. Listar todos os nomes empresariais de clientes e fornecedores (sem duplicar nomes).**

```sql
SELECT nome_empresarial FROM cliente
UNION
SELECT nome_empresarial FROM fornecedor;
```

---

### **2. Listar todos os e-mails de clientes e fornecedores, incluindo duplicados.**

```sql
SELECT email FROM cliente
UNION ALL
SELECT email FROM fornecedor;
```

---

### **3. Listar todos os CPFs de contactos e todos os CNPJs de clientes em um único resultado.**

```sql
SELECT cpf AS documento FROM contacto
UNION
SELECT cnpj AS documento FROM cliente;
```

---

### **4. Listar todos os telefones existentes nas tabelas cliente e fornecedor (sem duplicar).**

```sql
SELECT telefone FROM cliente
UNION
SELECT telefone FROM fornecedor;
```

---

### **5. Listar todos os IDs de cliente que aparecem na tabela lancamento e todos os IDs de fornecedor também presentes.**

```sql
SELECT id_cliente FROM lancamento
UNION
SELECT id_fornecedor FROM lancamento;
```

---

### **6. Listar todos os nomes das contas e nomes das categorias em uma única lista.**

```sql
SELECT nome FROM conta
UNION
SELECT nome FROM categoria;
```

---

### **7. Listar todos os estados presentes na tabela cliente e fornecedor (sem duplicar).**

```sql
SELECT estado FROM cliente
UNION
SELECT estado FROM fornecedor;
```

---

### **8. Listar todas as descrições de lançamentos e descrições das categorias, incluindo duplicados.**

```sql
SELECT descricao FROM lancamento
UNION ALL
SELECT descricao FROM categoria;
```

---

### **9. Listar todas as cidades cadastradas entre cliente e fornecedor.**

```sql
SELECT cidade FROM cliente
UNION
SELECT cidade FROM fornecedor;
```

---

### **10. Listar todos os registros inativos entre clientes e fornecedores.**

```sql
SELECT nome_empresarial, situacao FROM cliente WHERE situacao = 'INATIVO'
UNION
SELECT nome_empresarial, situacao FROM fornecedor WHERE situacao = 'INATIVO';
```
