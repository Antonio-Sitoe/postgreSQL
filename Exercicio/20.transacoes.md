## **Exercício 1 — Inserção de Cliente com Commit**

**Contexto:** Você precisa adicionar um novo cliente à base de dados.
**Objetivo:** Inserir um registro na tabela `cliente` dentro de uma transação e confirmá-lo.
**Tabelas envolvidas:** `cliente`
**Tarefa:**

1. Inicie uma transação.
2. Insira um cliente com todos os dados obrigatórios.
3. Faça `COMMIT` para salvar a transação.

## **Exercício 6 — Transferência de Saldo Entre Contas**

**Contexto:** Transferir valor entre contas bancárias.
**Objetivo:** Garantir atomicidade na transferência entre duas contas.
**Tabelas envolvidas:** `conta`
**Tarefa:**

1. Inicie uma transação.
2. Subtraia o valor da conta de origem.
3. Adicione o valor à conta de destino.
4. Use `COMMIT`.
5. Certifique-se de que se algum passo falhar, o saldo original seja mantido.

---

## **Exercício 7 — Inserção de Lançamento com Atualização de Conta**

**Contexto:** Registrar um lançamento financeiro e atualizar o saldo da conta correspondente.
**Objetivo:** Demonstrar transação envolvendo duas tabelas.
**Tabelas envolvidas:** `lancamento`, `conta`
**Tarefa:**

1. Inicie uma transação.
2. Insira um lançamento para um cliente.
3. Atualize o saldo da conta associada.
4. Use `COMMIT`.
5. Se o valor do lançamento for negativo, faça `ROLLBACK`.

---

## **Exercício 8 — Inserção de Múltiplos Lançamentos com Validação**

**Contexto:** Inserir várias parcelas de um mesmo lançamento.
**Objetivo:** Garantir que todas as parcelas sejam válidas antes de confirmar a transação.
**Tabelas envolvidas:** `lancamento`
**Tarefa:**

1. Inicie uma transação.
2. Insira 3 parcelas para o mesmo cliente.
3. Verifique se todos os valores são positivos.
4. Se algum valor for inválido, faça rollback; caso contrário, commit.

## **Exercício 10 — Alteração de Situação de Cliente**

**Contexto:** Um cliente mudou de status de ATIVO para INATIVO.
**Objetivo:** Demonstrar transação segura na alteração de situação de cliente.
**Tabelas envolvidas:** `cliente`
**Tarefa:**

1. Inicie uma transação.
2. Atualize a coluna `situacao` para 'INATIVO'.
3. Verifique se o cliente não possui lançamentos pendentes.
4. Se possuir, faça rollback; caso contrário, commit.
